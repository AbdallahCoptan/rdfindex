\begin{abstract}
The compilation of several key performance indicators in just one value is 
becoming a challenging task in certain domains to summarize data and information. 
Furthermore, policy makers are continuosly gathering and analyzing statistical 
data with the objective of providing objective measures about a specific policy, activity, 
product or service and making some kind of decission but existing tools 
and techniques based on traditional processes are preventing a 
proper use of the new dynamic and data environment avoiding more timely, 
adaptable and flexible (on-demand) quantitative index creation. In this context 
semantic-based technologies emerges to provide the adequate building blocks 
to represent and process domain-knowledge in a flexible fashion 
using a common and shared data model. That is why a RDF vocabulary designed on 
the top of the RDF Data Cube Vocabulary to model quantitative indexes 
is introduced in this paper. Moreover a Java and SPARQL based processor 
of this vocabulary is also outlined as a tool to exploit metadata structure and automatically 
perform the computation process of a quantitative index. Finally some discussion, 
conclusions and future work are also outlined.
\end{abstract}
